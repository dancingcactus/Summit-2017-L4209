Cohort Table

Allows you to see how people frequently return to your app under different conditions.

Anatomy of a Cohort Table

  1. Granularity - eg. Day, Week, Month
  2. Inclusion Metric - What does a user have to do to be included in that cohort (Column 4)
  3. Return Metric - What does the user have to do to be included as a return (Colummns 5)
  4. Included - The number of people included in the cohort for that time period
  5. Returned - The number of people who returned each time period

Example #1:

Of the people who launched my app month how many people are using it after 2 months?

Step #1: Add a Cohort Table to the workspace.

Step #2: Add launches as both the inclusion metric and the return metric then run the report

Step #3: Analyze the results

Fallout

Measure funnels and workflows precisely

Anatomy of Fallout

  1. Touchpoints Dropzone - Can add dimension items, metrics or segments
  2. Segment Dropzone - Compare fallout for multiple groups by adding segments
  3. Fallout Container - Lets you set whether funnel should happen in a visit or longer
  4. Show All Visits -- Should the fallout start with all visits (Everyone) or the first touchpoint

Flow

Discover how people use your app vs how you think they should use your app

Anatomy of Flow

  1. Entry Dimension - Start the flow from the first interaction with that visit/visitor
  2. Dimension or Item - Starts the flow from the top items of a dimension or single item and goes both ways
  3. Exit Dimension - Flows backward from the last interaction of a visit/visitor
  4. Flow Settings - Specifies if flow should be restricted to the same visit or should be across the life of the visitor
